Фронтенд — самый важный вопрос
Варианты именно под мой кейс (замена WordPress + минимализм + «что угодно можно добавить»):

Next.js 15 (App Router) + TypeScript

Почему:
SSR + SEO из коробки (критично для блога).
Можно делать и статические страницы, и динамические (blog post).
Легко добавить "веб-приложения" как отдельные страницы/компоненты.
В будущем легко переехать на Vercel/Netlify.

Структура сайта будет выглядеть так:

├── Home (hero + последние статьи + "Обо мне" + "Приложения")
├── /blog
├── /blog/[slug]
├── /about
├── /apps (или /portfolio)
└── /admin (позже Keycloak)

Всё общается только с http://localhost:8080 (Gateway) — никаких прямых обращений к сервисам.
Изображения: content://... или просто /content/{uuid} через Gateway.

Как начать с нуля:

npx create-next-app@latest typo-front --typescript --tailwind --eslint --app
Поставь axios или fetch.
Сделай 3 страницы: Home, BlogList, BlogPost.
Всё.

Альтернативы (если Next.js пока страшно):

Простой вариант (1 день): Plain HTML + Tailwind + HTMX. Всё в папке gateway-server/src/main/resources/static. Никакого отдельного фронта. Работает сразу.
Thymeleaf в отдельном web-service — но это будет "монолит в микросервисах", не советую.
Vue/Nuxt — тоже ок, но Next.js лучше для SEO.

План:
Сначала довести бэк до состояния "я могу создать статью с обложкой через Postman и увидеть её на /blog".
Потом сделать Next.js фронт.
