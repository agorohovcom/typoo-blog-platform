Typoo Blog Platform - Архитектура и документация проекта
🎯 Общее описание
Typoo Blog Platform - это микросервисная блог-платформа, разрабатываемая как pet-project для замены WordPress на agorohov.com. Проект представляет собой монорепозиторий на Java с современным стеком технологий, ориентированный на масштабируемость и лучшие практики разработки.

Основная цель: Создание полноценной блог-платформы с возможностью расширения до полноценного веб-сайта с портфолио, информационными разделами и интерактивными приложениями.

🏗️ Архитектурные принципы
Микросервисная архитектура:
Монорепозиторий с модульной структурой

Service Discovery через Consul

API Gateway для маршрутизации запросов

Centralized Config с Spring Cloud Config

Event-Driven коммуникация через Kafka (в планах)

Технологический стек:
Java 21 + Spring Boot 3.5.3 + Spring Cloud 2025.0.0

Gradle с версионным каталогом (libs.versions.toml)

PostgreSQL + Liquibase для миграций

Docker + Docker Compose для контейнеризации

Testcontainers для интеграционного тестирования

🔧 Ключевые сервисы
1. Config Server (config-server)
   Централизованное управление конфигурациями

Поддержка профилей (dev, docker, prod)

Интеграция с Consul для service discovery

2. Gateway Server (gateway-server)
   Единая точка входа для всех запросов

Маршрутизация на основе service discovery

Load balancing, rate limiting (в планах)

Актуатор + метрики Prometheus

3. Article Service (article-service)
   MVP ядро блога - CRUD операций со статьями

Поддержка статусов (DRAFT, PUBLISHED, ARCHIVED)

Интеграция с PostgreSQL + Liquibase

Готовность к расширению (комментарии, теги)

4. Shared Module (shared)
   DotenvLoader - универсальный загрузчик переменных окружения

Поддержка профилей (.env.{profile})

Docker secrets integration

Базовые утилиты и DTO

🔄 Workflow разработки
Локальный запуск:
bash
# Развертывание инфраструктуры
docker-compose up -d consul postgres prometheus grafana

# Сборка и запуск сервисов
./gradlew clean build
docker-compose up --build config-server gateway-server article-service
Профили конфигурации:
dev - разработка с локальными .env файлами

docker - продакшен-подобное окружение

prod - будущая продакшен конфигурация

Тестирование:
bash

# Интеграционные тесты с Testcontainers
./gradlew :services:article-service:test
