# Typoo Blog Platform - Архитектура и документация проекта

## 🎯 Общее описание
Typoo Blog Platform - это микросервисная блог-платформа, разрабатываемая как pet-project для замены WordPress на agorohov.com. Проект представляет собой монорепозиторий на Java с современным стеком технологий, ориентированный на масштабируемость и лучшие практики разработки.

**Основная цель:** Создание полноценной блог-платформы с возможностью расширения до полноценного веб-сайта с портфолио, информационными разделами и интерактивными приложениями.

## 🏗️ Архитектурные принципы
**Микросервисная архитектура:**
- Монорепозиторий с модульной структурой
- Service Discovery через Consul
- API Gateway для маршрутизации запросов
- Centralized Config с Spring Cloud Config
- **gRPC для межсервисной коммуникации**
- Event-Driven коммуникация через Kafka (в планах)

**Технологический стек:**
- Java 21 + Spring Boot 3.5.3 + Spring Cloud 2025.0.0
- Gradle с версионным каталогом (libs.versions.toml)
- PostgreSQL + Liquibase для миграций
- **gRPC + Protocol Buffers** для API между сервисами
- Docker + Docker Compose для контейнеризации
- Testcontainers для интеграционного тестирования

## 🔧 Ключевые сервисы

### 1. Config Server (`config-server`)
- Централизованное управление конфигурациями
- Поддержка профилей (dev, docker, prod)
- Интеграция с Consul для service discovery

### 2. Gateway Server (`gateway-server`)
- Единая точка входа для всех запросов
- Маршрутизация на основе service discovery
- Load balancing, rate limiting (в планах)
- Актуатор + метрики Prometheus

### 3. Article Service (`article-service`)
- MVP ядро блога - CRUD операций со статьями
- Поддержка статусов (DRAFT, PUBLISHED, ARCHIVED)
- Интеграция с PostgreSQL + Liquibase
- Поддержка тегов, категорий, версионирования
- **Интеграция с Content Service через gRPC**

### 4. Content Service (`content-service`) - **НОВЫЙ**
- Универсальный сервис для управления файлами
- Поддержка различных хранилищ (локальное, S3, GridFS)
- Обработка изображений (ресайз, оптимизация)
- gRPC API для внутренней коммуникации
- REST API для фронтенда через Gateway
- Категории файлов: статьи, портфолио, аватары, общие

### 5. Shared Module (`shared`)
- DotenvLoader - универсальный загрузчик переменных окружения
- Поддержка профилей (`.env.{profile}`)
- Docker secrets integration
- Базовые утилиты и DTO
- Общие gRPC stubs и protobuf контракты

## 🚀 Особенности реализации

### Content Service Architecture
```protobuf
// Пример gRPC контракта
service ContentService {
  rpc UploadFile(UploadFileRequest) returns (FileResponse);
  rpc GetFile(GetFileRequest) returns (stream FileChunk);
  rpc ValidateFiles(ValidateFilesRequest) returns (ValidateFilesResponse);
}

// URI схема для ссылок на файлы в статьях
content://files/{uuid}                    # Прямая ссылка
content://images/{uuid}?variant=thumbnail # Варианты изображений
```

### DotenvLoader System
```java
// Автоматическая загрузка переменных по профилю
DotenvLoader.loadEnvironmentVariables(environment)

// Поддерживаемые пути:
// - services/{service}/.env.{profile} (dev)
// - /run/secrets/{service}_env_file (docker)
// - Системные переменные (высший приоритет)
```

### Gradle Version Catalog
```toml
# Единое управление зависимостями
[versions]
spring-boot = "3.5.3"

grpc-spring-boot = "3.1.0.RELEASE"

[libraries]
grpc-starter = { module = "net.devh:grpc-spring-boot-starter" }
protobuf-java = { module = "com.google.protobuf:protobuf-java" }
```

## 🔄 Workflow разработки

### Локальный запуск:
```bash
# Развертывание инфраструктуры
docker-compose up -d consul postgres prometheus grafana

# Сборка и запуск сервисов
./gradlew clean build
docker-compose up --build config-server gateway-server article-service content-service
```

### Профили конфигурации:
- `dev` - разработка с локальными .env файлами
- `docker` - продакшен-подобное окружение
- `prod` - будущая продакшен конфигурация

### Тестирование:
```bash
# Запуск всех тестов
./gradlew clean test

# Интеграционные тесты с Testcontainers
./gradlew :services:article-service:test
```

## 🎯 Уникальные особенности

### 1. Гибкая система конфигурации
- Единый DotenvLoader для всех сервисов
- Поддержка Docker secrets для безопасности
- Приоритеты: System Properties > Env Vars > .env files

### 2. Универсальный Content Service
- **Абстракция над хранилищем** (локальное/S3/GridFS)
- **gRPC для эффективной коммуникации**
- Поддержка вариантов изображений (thumbnail, medium, large)
- Единая точка управления файлами для всех сервисов

### 3. Оптимизированная сборка
- Кэширование зависимостей Gradle в Docker
- Мульти-модульная структура с shared-библиотекой
- Типобезопасный доступ к проектам (TYPESAFE_PROJECT_ACCESSORS)

### 4. Production-ready инфраструктура
- Health checks для всех сервисов
- Мониторинг метрик через Micrometer + Prometheus
- Готовность к горизонтальному масштабированию

### 5. Event-Driven готовность
- Архитектура готова к внедрению Kafka
- Спроектирована для CQRS и Event Sourcing
- Возможность асинхронной обработки комментариев/уведомлений
